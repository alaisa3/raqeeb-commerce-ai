# رقيب التجارة | Raqeeb Commerce AI

**رقيب التجارة** هو إيجنت لوجستي وذكي متكامل يراقب شحنات متجرك بـ **سلة (Salla)**، ويرصد تلقائيًا تأخّر التوصيل، ويتنبّأ بالمشاكل قبل تحوّلها إلى شكاوى رسمية!

---

## 🇸🇦 مميزات المنصة الرئيسية (SaaS Core Features)

1. **مراقبة الشحنات المتأخرة والراكدة:** رصد حي وفوري للطرود المعلقة لدى شركات الشحن الكبرى (سمسا، أرامكس، ناقل، SPL، DHL).
2. **تحليل ذكي وعالي الدقة لمستوى الخطورة:** تصنيف الطلبات إلى منخفضة ومتوسطة وعالية الخطورة بناءً على تكرار تأخر الناقل ونوع المنطقة الجغرافية في المملكة.
3. **نسخ مسودات واتساب بلمسة زر:** تجهيز ردود اعتذارية واحترافية مخصصة للعملاء مع صياغة رسائل تصعيد قوية لشركات الشحن باستخدام ٣ نبرات (رسمية، ودية، ومختصرة)، مع دعم اختياري لـ **Gemini AI** لتحسين النبرة.
4. **بوابة إدارة المرتجعات بقواعد مرنة:** إصدار قرارات قبول أو رفض مبدئية تلقائية للمنتجات بناءً على حالة المنتج وغلافه ونافذة الإرجاع القانونية لحماية هوامش أرباح التاجر.
5. **لوحة تحكم إدارية ومؤشرات أداء (KPIs):** عرض إحصائيات معالجة الطلبات اليومية، التنبيهات المفتوحة، وسجل محاكاة العمليات اللحظي (Live Audit Logs).
6. **دعم استيراد البيانات اليدوية (CSV):** إمكانية لصق أو رفع ملفات الطلبات وحساب نسب الأمان بالكامل دون ربط مباشر.

---

## 🛠️ التقنيات المستخدمة (Tech Stack)

- **جهة العميل (Frontend):** React (مع طراز Cairo الأنيق للخطوط العربية)، Tailwind CSS، ورموز Lucide-React.
- **الخادم الخلفي (Backend):** Node.js, Express, TypeScript, tsx, and esbuild.
- **التكامل الذكي (AI Engine):** `@google/genai` (SDK الحديث لتشغيل ومحاكاة Gemini 3.5 Flash).
- **إدارة البيانات المتكاملة:** قاعدة بيانات محلية مرنة مشحونة مسبقاً لحفظ تفضيلات وسياسات متجرك تلقائياً (`database.json`).

## ⚙️ متغيرات البيئة المطلوبة (Environment Variables)

قم بإنشاء ملف `.env` في المسار الرئيسي وتعيين المتغيرات التالية:

```env
# مفتاح منصة جوجل للذكاء الاصطناعي (اختياري، يتم تحسين الرسائل به فورياً)
GEMINI_API_KEY="YOUR_GEMINI_API_KEY"

# بيانات ربط منصة سلة السعودية للتجارة (Salla OAuth 2.0 Credentials)
SALLA_CLIENT_ID="YOUR_SALLA_CLIENT_ID"
SALLA_CLIENT_SECRET="YOUR_SALLA_CLIENT_SECRET"
SALLA_REDIRECT_URI="https://your-domain.com/api/salla/callback"
SALLA_AUTH_URL="https://accounts.salla.sa/oauth2/auth"
SALLA_TOKEN_URL="https://accounts.salla.sa/oauth2/token"
SALLA_API_BASE_URL="https://api.salla.dev/admin/v2"

# رابط التطبيق الأساسي
APP_URL="http://localhost:3000"
```

## 🚀 طريقة التشغيل محلياً (How to Run Locally)

1. قم بتثبيت مصفوفة المكتبات المطلوبة:
   ```bash
   npm install
   ```

2. ابدأ تشغيل الخادم والمنصة بالوضع التجريبي والتفاعلي:
   ```bash
   npm run dev
   ```

3. افتح المتصفح على الرابط المباشر:
   `http://localhost:3000`

---

## 🌐 تفاصيل النشر والإنتاج (Production Deployment)

التطبيق مهيأ بالكامل للبناء السريع ومتوافق مع حلول الحوسبة السحابية وVercel:

- عند تشغيل `npm run build`:
  - يتم تجميع واجهة الويب في مجلد `dist/`.
  - يقوم esbuild بتجميع ملف الخادم `server.ts` بإنتاجية فائقة السرعة إلى ملف خفيف ومغلف `dist/server.cjs`.

- عند تشغيل `npm start`:
  - يتم استضافة الواجهة وخادم العمليات عبر المنفذ الافتراضي الموحد 3000 تلقائياً.

## 🎁 طريقة تشغيل الوضع التجريبي (Demo Mode Instructions)

لتمكين تجربة فائقة من اللمسة الأولى ودون الحاجة لتسجيل حساب سلة مباشر:
1. المنصة مدعومة فورياً بـ **البيانات السعودية التجريبية** لمدن (الرياض، مكة، جدة، تبوك، أبها).
2. انقر على زر **"استعادة البيانات الافتراضية"** بأعلى لوحة التحكم لتعبئة السجلات فوراً بطلبات ذات مخاطر متفاوتة ومرتجعات قيد المراجعة.
3. يمكنك اختبار جهارة المزامنة بالنقر على زر **"مزامنة طلبات سلة"** لمحاكاة ورود طلب جديد بالكامل من خوادم سلة.
